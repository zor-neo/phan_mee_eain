# Phan Mee Eain Internal Team Tester Checklist

Use this checklist to test the website like a normal user and internal team member. You do not need technical knowledge.

Please test slowly, write what happened, and take screenshots if something looks wrong.

Do not write passwords in this document. Use the test passwords shared separately by the project team.

## Tester Information

```text
Tester name:
Date:
Device: Desktop / Laptop / Tablet / Phone
Browser: Chrome / Edge / Firefox / Safari / Other
Internet connection: Good / Average / Slow
Test account email:
```

## How To Mark Results

Use one mark for each item.

```text
OK = works as expected
Issue = something is wrong
Skip = not tested
```
